# Exercices Java — Classes et objets (POO – bases)

## Exercice 1 — Compte bancaire

(Bases : classe, attributs, méthodes)

## Objectif

Créer une classe Java représentant un compte bancaire simple et manipuler un objet.

## Contexte

Tu développes un programme qui modélise un compte bancaire avec des opérations basiques.

## Travail demandé

### 1. Classe

- Créer une classe nommée **CompteBancaire**

### 2. Attributs

La classe doit contenir :

- le nom du titulaire
- le numéro du compte
- le solde

Choisir des types adaptés et respecter les conventions Java.

### 3. Méthodes

Ajouter des méthodes permettant de :

- afficher les informations du compte
- déposer de l’argent
- retirer de l’argent

Les méthodes doivent utiliser et modifier les attributs.

### 4. Programme principal

Dans la méthode `main` :

- créer au moins un objet `CompteBancaire`
- appeler ses méthodes
- afficher les résultats dans la console

## Contraintes

- pas de tableau
- pas de boucle
- pas de logique complexe
- focus uniquement sur classe et objet

## Exercice 2 — Étudiant et résultats

(Constructeur et méthodes)

## Objectif

Comprendre le rôle du constructeur et l’initialisation des objets.

## Contexte

Tu représentes un étudiant et ses résultats scolaires.

## Travail demandé

### 1. Classe

- Créer une classe nommée **Etudiant**

### 2. Attributs

La classe doit contenir :

- le nom
- le prénom
- la note
- un indicateur d’admission (oui / non)

### 3. Constructeur

- Le constructeur doit recevoir les informations principales
- Il initialise les attributs de l’objet

### 4. Méthodes

Ajouter des méthodes pour :

- déterminer si l’étudiant est admis
- afficher le bulletin de l’étudiant

### 5. Programme principal

Dans `main` :

- créer plusieurs objets `Etudiant`
- afficher leurs résultats

## Contraintes

- pas de tableau
- pas de boucle
- pas de Scanner
- constructeur obligatoire

## Exercice 3 — Produit et panier

(Interaction entre objets)

## Objectif

Manipuler plusieurs objets et comprendre leur interaction.

## Contexte

Tu développes une simulation simple de panier d’achat.

## Travail demandé

### 1. Classe Produit

Créer une classe **Produit** avec :

- nom du produit
- prix
- quantité

### 2. Constructeur

- Le constructeur initialise tous les attributs

### 3. Méthodes

Ajouter des méthodes pour :

- calculer le prix total du produit
- afficher les informations du produit

### 4. Programme principal

Dans `main` :

- créer plusieurs objets `Produit`
- afficher leurs informations
- afficher le coût total du panier

## Contraintes

- pas de tableau
- pas de boucle
- pas de notions POO avancées (héritage, polymorphisme)
- focus sur des objets simples et concrets

## Résumé pédagogique

- Exercice 1 : fondations de la POO
- Exercice 2 : constructeur et logique simple
- Exercice 3 : interaction entre objets
